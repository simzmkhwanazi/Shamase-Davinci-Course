Q001 · Module 01
A client commissions a 12-minute corporate documentary, deliverable in 10 days. You are fluent in Premiere Pro but have never used Resolve. The client requires only an H.264 4K master and an Instagram-ready vertical cut.
What is the most professionally responsible decision?
ASwitch to Resolve to learn it on a real project — the free version handles everything required.
BDeliver the project in Premiere Pro and begin learning Resolve on a lower-stakes future project.
CUse Resolve for the edit and Premiere for the colour grade to play to each app's strengths.
DSubcontract the entire job to a Resolve specialist while you observe.
✓ WHY B IS CORRECT
The professional argument is risk management. A 10-day deadline on a deliverable that doesn't need Resolve isn't the right time to learn a new tool. Use the skills you have; learn the new tool on a lower-stakes project.
Example. It's like being a fluent Premiere editor handed a tight deadline — you deliver in the tool you already know and learn Resolve on next month's lower-stakes personal short.
Why the other options are incorrect
AResolve is technically capable, but learning curve + deadline = unnecessary risk on a paid job.
CMixing tools mid-project compounds risk: round-tripping problems, two software workflows, two debugging surfaces.
DSubcontracting is overkill for a 12-minute corporate piece; it shifts the work but adds coordination overhead.
Principle: Match the tool to the project, not the project to the tool you want to learn.
