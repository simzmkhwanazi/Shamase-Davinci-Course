# Fairlight Audio Certification — Shamase Studios

A self-study certification program for audio post-production with DaVinci Resolve's
**Fairlight** page, built from the official Blackmagic "DaVinci Resolve Fairlight
Training" series (Mary Plummer). Part of the Shamase Studios post-production
curriculum.

## What's here

| File | Purpose |
|------|---------|
| `index.html` | Certification landing page — requirements, mandatory screenshot acknowledgement gate, links. |
| `study-guide.html` | 7-module written curriculum + 70 interactive knowledge-check questions. |
| `exam1.html` | **Exam 1 — Fundamentals**: 100 interface/terminology questions. |
| `exam2.html` | **Exam 2 — Workflow & Decisions**: 100 scenario questions. |
| `certificate.html` | Printable certificate, unlocked only after passing **both** exams. |
| `_build/` | Python source that generates the exam/study-guide HTML (see below). |

## Curriculum (7 modules)

1. Introduction to Audio & the Fairlight Page
2. Editing Dialogue
3. Sound Design & Effects Editing
4. Recording Voiceover & ADR
5. Mixing
6. Repairing & Restoring Audio
7. Delivering & Mastering

## Certification rules

- Two exams, **100 questions each**, 4 options, one best answer.
- **2-hour** timer per exam, auto-submit at time-up.
- **90%** pass mark, **3 attempts** maximum per exam (tracked in `localStorage`).
- Pass **both** exams to unlock the printable certificate.
- Candidates must screenshot their results page and send it to their instructor.

## Rebuilding / extending

The exam pages reuse the proven exam engine from `../02-mock-exam-basics.html`.
Content lives in `_build/` as Python data; the build clones the engine and injects it.

```bash
cd _build
python3 build.py      # regenerates exam1.html, exam2.html, study-guide.html
```

- `data_exam1.py` — Exam 1 question bank (fundamentals).
- `data_exam2.py` — Exam 2 question bank (scenarios).
- `data_studyguide.py` — study-guide content + knowledge checks + renderer.
- `build.py` — clones the engine template, injects the banks and config, writes the HTML.

The landing page (`index.html`) and `certificate.html` are hand-authored and not
generated by the build.

## Notes

- Built against Resolve 17–20 (Fairlight fundamentals are consistent across these).
- No backend / no database — everything runs in the browser; GitHub Pages compatible.
