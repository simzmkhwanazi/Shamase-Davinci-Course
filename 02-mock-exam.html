SECTION I · MODULE 01Foundations & Philosophy

8 questions on the post-production workflow, the seven pages, Free vs Studio, and project library management.
001

A client commissions a 12-minute corporate documentary, deliverable in 10 days. You are fluent in Premiere Pro but have never used Resolve. The client requires only an H.264 4K master and an Instagram-ready vertical cut.What is the most professionally responsible decision?

  1. Switch to Resolve to learn it on a real project — the free version handles everything required.
  2. Deliver the project in Premiere Pro and begin learning Resolve on a lower-stakes future project.
  3. Use Resolve for the edit and Premiere for the colour grade to play to each app's strengths.
  4. Subcontract the entire job to a Resolve specialist while you observe.
002

You're spec'ing a workstation that will primarily handle 4K H.265 multicam interviews, light colour, no Fusion, and Fairlight mixes up to 24 tracks.Which hardware spec is the weakest link to address first?

  1. CPU clock speed — H.265 decode is single-thread limited.
  2. GPU VRAM — Resolve needs at least 12 GB for 4K colour work.
  3. RAM — multitrack Fairlight projects load all audio into memory.
  4. Storage I/O — H.265 multicam needs fast random reads to scrub smoothly.
003

A colleague claims Resolve's free version cannot be used for commercial work because of licensing restrictions. How do you respond?

  1. They are correct — the free version is for personal projects only.
  2. They are incorrect — the free version has no commercial-use restriction; Studio is bought for features, not licensing.
  3. They are partially correct — commercial use is permitted up to a revenue cap, beyond which Studio is required.
  4. They are partially correct — commercial use is permitted, but client redelivery requires Studio.
004

A client sends you a .drp file and asks you to "open the project." Double-clicking launches Resolve but the project does not open. What is happening?

  1. The .drp is corrupted; ask the client to re-export.
  2. A .drp is a project export, not a project file — you must import it into a library via the Project Manager.
  3. The free version cannot open .drp files from Studio.
  4. The .drp requires a matching .drx to open.
005

You must move a 90-minute project from your Mac to a colleague's PC running Resolve free for a single grading session.The cleanest export strategy is:

  1. Copy your entire Resolve library folder and tell them to point Resolve at it.
  2. Right-click the project → Export Project → send the .drp plus a hard drive with the linked media.
  3. Render a high-quality H.265 file and have them reimport it.
  4. Save to a cloud library so the colleague accesses it directly.

A Studio-only effect stops working in a project that was briefly opened on a colleague's free version. The colleague returned it to you and the effect is locked. What's true?

  1. The effect was deleted automatically by the free version.

In a typical post workflow, which Resolve page is used last?

  1. Color — colour is the final creative pass.

  3. Deliver — exports the final file once all other work is complete.

A junior editor asks whether to learn Cut or Edit first. Their goal is narrative film.The best answer is:

  1. Cut first — it's simpler and Edit extends its concepts.

010

The practical distinction between Media Storage and the Media Pool is:

  1. Media Storage holds camera files; Media Pool holds transcoded copies.
  2. Media Storage is the file system browser; Media Pool is the project's database of clips Resolve tracks.

  4. On the Cut page, set both cameras to the Sync Bin and let Resolve auto-pick the best audio.

A wide shot ends with the actor turning right; a close-up take also begins with them turning right. Cutting at the motion feels natural because:

  1. The viewer's eye is occupied by motion, masking the discontinuity between angles.

022

Which keyboard setup most accelerates dialogue and podcast editing once learned?

  1. Defaults: Ctrl+\ split, Ctrl+Shift+[/] for trim.
  2. Custom: Q = trim start, W = trim end, S = split. Three fingers, one hand.
  3. The Premiere preset for muscle-memory transfer.
  4. The Final Cut Pro preset.
023

You keyframe a title sliding in from offscreen-right to centre. The animation works but jerks to a stop.Cause:

  1. Keyframes are linear by default — ease the outgoing keyframe by flattening its tangent in the Keyframes panel.
  2. The title is too short — animations under 24 frames always jerk.
  3. Resolve doesn't support keyframed titles on the Edit page — use Fusion.
  4. The Inspector transform applies only once at clip start.
024

You keyframed a clip's position to move across screen. You duplicate the clip. The duplicate must animate identically. Most reliable approach:

  1. The duplicate inherits the animation — just duplicate normally.
  2. Use Edit → Copy → Paste Attributes onto the duplicate.
  3. Recreate keyframes manually on the duplicate.
  4. Create a Compound Clip from the original first — duplicating compounds preserves animation.
025

Enabling Snapping (the magnet icon) in the Edit page affects what?

  1. Clips snap to nearby edit points, markers, and the playhead while dragging.
  2. The viewer snaps to common aspect ratios.
  3. Audio waveforms snap to even peaks for sync.
  4. Clips snap to frame boundaries to prevent sub-frame edits.
026

A clip has good dialogue then 4 seconds of crew noise. You want to keep the dialogue and replace the noisy audio (only) with clean wild track audio. Video must stay intact.

  1. Split the clip at the noise boundary, delete the noisy half, ripple-trim earlier content right.
  2. Unlink V/A (chain icon), then on audio only: split, delete the noisy section, replace with the wild track.
  3. Apply Audio FX → Noise Reduction on the noisy section.
  4. Cut the whole video clip and re-edit from the source viewer's audio-only mode.
027

Simplest way to fade in a generator title:

  1. Right-click the clip → Add Transition → Cross Dissolve.
  2. Drag the small white triangle handle at the clip's top-left corner inward.
  3. Apply a Fade-In effect from the Effects panel.
  4. Keyframe opacity from 0 to 100% in the Inspector.
028

A 1-hour interview timeline. You need to render only minutes 12 to 18 for client preview without destroying timeline integrity.

  1. Duplicate the timeline, delete everything outside that range, render the duplicate.
  2. Set in/out (I/O) around the range, switch to Deliver, set render range to "In to Out."
  3. Use the Cut page to chop out a sub-timeline, deliver from it.
  4. Export the whole timeline as H.264 then trim the resulting file externally.

  4. Background generator (grey) → Merge over MediaIn with composite Multiply.

You want to view one node in the left viewer while the final composite stays in the right viewer. Fastest method:

SECTION VII · MODULE 07The Fairlight Page

10 questions on Fairlight workflow, tracks, buses, dynamics, and effects.
057

You can hear your music tracks but the dialogue is barely audible. You add a Compressor to the dialogue. Threshold is set to -10 dB, ratio 4:1, makeup gain 0 dB. Why is the dialogue still quiet?

  1. Makeup gain must be positive to compensate for the compression — try +6 dB.
  2. The Compressor only affects audio above -10 dB; if dialogue averages below that, no compression and no makeup happens.
  3. The dialogue track is muted — check the mute button.
  4. The Compressor cannot work on dialogue tracks; use Limiter instead.
058

A character's voice is heard from another room through a closed door.The audio treatment that best sells the "in another room" feel:

  1. Increase volume on that track.
  2. Roll off high frequencies with an EQ (high-cut), and reduce overall track volume slightly.
  3. Add chorus and reverb generously.
  4. Apply a flanger effect.
059

An audio recording imports as a stereo clip but contains the voice only on the left channel. The right approach:

  1. Pan the stereo track left to centre the voice — leave it stereo.
  2. Right-click in Media Pool → Clip Attributes → Audio → set format to mono / both channels. Then on the timeline track header, right-click → Change Track Type → Mono.
  3. Duplicate the clip and stack copies to fake mono.
  4. Use Fairlight's "channel rebalance" plugin.
060

A scene has 5 body-drop sound effects on 5 separate tracks. You want to apply consistent compression and reverb across all 5 with one set of controls. The right structure:

  1. Apply compression and reverb individually to each track — Fairlight is per-track.
  2. Group the 5 tracks (Ctrl+G) and apply effects to the group.
  3. Route all 5 tracks to a single Bus, apply compression and reverb on the Bus.
  4. Render all 5 to a single audio file and apply effects to that.
061

In the channel strip, the green strip near the top opens what?

  1. Equalizer.
  2. Dynamics — Compressor, Gate, Expander, Limiter.
  3. Bus routing.
  4. Effects rack.
062

A 30-second music clip needs to fit a 20-second sequence on beat. The non-AI approach to shorten it:

  1. Decrease the timeline frame rate.
  2. Find similar waveform peaks at the start and end of a section you want to keep, blade at those points, splice, add a short crossfade at the splice.
  3. Apply a time-stretch effect with 67% rate.
  4. Render and re-import at a faster speed.
063

In Range mode on the Fairlight timeline, dragging across part of an audio clip does what?

  1. Selects only that range — backspace deletes it; you can apply effects or pull volume keyframes inside it without splitting the clip.
  2. Splits the clip at both ends of the drag.
  3. Creates a new audio track for that range.
  4. Moves the audio waveform vertically.
064

The dialogue audio level varies wildly between very loud and very quiet across one clip. Quickest path to consistent perceived loudness without re-recording:

  1. Manually keyframe volume on every change.
  2. Apply a Compressor with the Dialog Compression preset; refine threshold/ratio so the gain-reduction meter shows -3 to -6 dB on peaks.
  3. Apply a Limiter set to 0 dB.
  4. Increase the track fader to maximum.
065

Three audio tracks (dialogue, music, SFX) all route to Bus 1 (main mix). You want all dialogue to share one EQ but not the music or SFX. The correct route:

  1. Add EQ to Bus 1 — it filters dialogue only by frequency.
  2. Create a Dialogue Bus, route dialogue tracks there, route the Dialogue Bus to Bus 1, apply EQ on the Dialogue Bus.
  3. Apply EQ to each dialogue clip individually.
  4. Group the dialogue tracks and apply EQ to the group.
066

A new bus is created (Fairlight → Bus Format) but nothing routed to it plays through. Most likely cause:

  1. The bus has no output assigned — it must be routed to Bus 1 or a final output.
  2. The bus is muted by default.
  3. Buses cannot carry audio in the free version.
  4. The sample rate is wrong.

SECTION VIII · MODULE 08The Deliver Page

10 questions on render settings, codec choice, queue management, and delivery edge cases.
067

A client wants a master file to archive that they can re-edit in five years if needed. The right codec choice is:

  1. H.264 — universally compatible.
  2. H.265 — smallest file size.
  3. ProRes 422 HQ or 4444 — large but visually pristine and easy to decode in any future NLE.
  4. QuickTime Animation — lossless.
068

A 1080p edit will be uploaded to YouTube. The "trick" to make it look noticeably cleaner after YouTube's re-encoding:

  1. Render in H.265 instead of H.264.
  2. Upscale the deliver resolution to Ultra HD (3840×2160) — YouTube applies a different encoder to 4K uploads, improving final quality.
  3. Render at 10-bit colour depth.
  4. Increase the bitrate to 50 Mbps.
069

A client asks for an MP4 video file. The container is MP4. What codec is conventionally used inside it?

  1. ProRes.
  2. H.264 (sometimes H.265) — these are the typical codecs paired with MP4.
  3. DNxHR.
  4. QuickTime Animation.
070

After hitting Render, the rendered file plays back with white blocks appearing randomly across the image. Most likely cause:

  1. The codec is corrupted.
  2. The Mac is overheating during render and skipping frames — render in a cooler environment with better airflow.
  3. The deliver resolution is too high for the source.
  4. Resolve free is rate-limited at low frame rates.
071

You add a subtitle track in the Edit page using the Subtitle generator. After delivery the subtitles do not appear in the output file. Most likely cause:

  1. The subtitle format wasn't set to "Burn into video" in the Deliver page subtitle settings.
  2. The subtitle clip is on a separate timeline.
  3. Subtitles only render in H.265.
  4. The subtitle track was muted.
072

You need to render the same timeline as three deliverables: a YouTube version, a client preview, and a ProRes archive. You want them all to render unattended overnight.The right approach:

  1. Open three Resolves and start each render in parallel.
  2. Configure each as a separate render job, Add to Render Queue, then Render All — they process sequentially.
  3. Render manually one at a time during the day.
  4. Use the Cut page's batch export.
073

For an archive master, the audio settings should be:

  1. AAC, stereo, 16-bit — universally compatible.
  2. Linear PCM (WAV-style), 32-bit float if available, with "All Timeline Tracks" rather than a stereo mixdown.
  3. MP3 320 kbps.
  4. Whatever the input audio was, unchanged.

  3. DCI-P3.
  4. Rec.2020.

SECTION X · MODULE 10The Node Tree Architecture

8 questions on the Notary Structure and node types.
085

The three conceptual columns of the Notary Structure, in order:

  1. Look — Balance — Isolate.
  2. Balance — Isolate — Global Look.
  3. Isolate — Balance — Sharpen.
  4. Curves — Wheels — Effects.
086

A Serial node connection means:

  1. Multiple corrections operate on the same source image simultaneously.
  2. Each node operates on the previous node's output — order matters.
  3. Connections render in the background.
  4. The two nodes share a single set of controls.
087

A Parallel Mixer node combines its inputs how?

  1. Bottom branches override top branches (priority order).
  2. All branches start from the same source and sum equally — no precision compounding loss.
  3. Branches multiply together.
  4. The mixer picks the most-saturated branch at each pixel.
088

A Layer Mixer node combines its branches how?

  1. Equally summed.
  2. Bottom branches take priority — they sit on top, like Photoshop layers.
  3. Branches are blended by image hue.
  4. The brightest branch wins per pixel.
089

In the Notary Structure, the Main Object node is wired as the bottom branch of the Layer Mixer because:

  1. Layer mixers process bottom-first.
  2. The Main Object should take priority — corrections applied to the background must not affect it. Bottom branches override top branches in a Layer Mixer.
  3. It connects naturally to the CST out.
  4. Convention only — order doesn't actually matter.
090

A grade involves: exposure correction, contrast adjustment, blue tint, skin isolation, vignette, and final film grain. The right distribution across node types:

  1. Everything in serial nodes — simplest.
  2. Balance (exposure, contrast) in serial; skin isolation via Layer Mixer; tint and vignette via Parallel Mixer; grain in a final serial node.
  3. Everything in one parallel mixer.
  4. One Layer Mixer per correction.
091

A custom keyboard shortcut for "Label Selected Node" is most useful because:

  1. It makes the node graph print more legibly.
  2. Naming nodes as you work (with F2) preserves understanding when you return to the project days later — you can see what each node is for.
  3. It enables Studio-only features.
  4. Labels render automatically into the export.
092

A power grade (.drx file) imported into the Gallery's Power Grades folder lets you:

  1. Apply a complete pre-built node tree to any clip with one click — the entire Notary Structure can be inherited.
  2. Lock the grade so it can't be modified.
  3. Convert the grade to a single LUT.
  4. Apply only the colour wheel positions, not the node structure.

8 questions on exposure, contrast, saturation, and the role of balancing.
093

The Parade scope shows the waveform of your image bunched in the lower third (everything dark) with no signal reaching the top. The right initial move:

  1. Push gain up to extend highlights toward 1023, then refine gamma to lift midtones.
  2. Increase saturation.
  3. Apply a sharpening filter.
  4. Change the Output Color Space.
094

The top of the waveform shows a flat horizontal line at 1023. This indicates:

  1. Perfect exposure.
  2. Highlights are clipped — pixel values exceeding 1023 are all recorded as 1023, and the original detail is lost.
  3. The image is over-saturated.
  4. The codec is failing.
095

A "muddy" image — skin tones feel dark and blotchy. The cleanest fix in the Contrast node:

  1. Apply heavy global contrast.
  2. Apply moderate contrast with the pivot pulled left, so blacks deepen without crushing midtones.
  3. Apply a Glow effect.
  4. Push saturation up.
096

At the balancing stage, the right amount of saturation to add is:

  1. Maximum — push to full so the grade has plenty to work with.
  2. Modest — just enough that the qualifier can distinguish regions (e.g. skin from sky).
  3. Zero — desaturate completely until the global look.
  4. None of the above — saturation isn't part of balancing.
097

A scene was shot in a candle-lit room and reads as warm orange throughout.Should you white-balance it to neutral?

  1. Yes — every scene must be neutralised first.
  2. No — balancing fixes errors, not character. Candle light is warm by nature; leave the warmth as the scene's identity.
  3. Only if delivering for broadcast.
  4. Only if there's skin in the frame.
098

In Studio, the HDR Wheels palette differs from standard Primaries by:

  1. Six tonal regions instead of three (lift/gamma/gain), each with independent thresholds.
  2. HDR-only colour space support.
  3. Required for 4K work.
  4. Different colour wheel mathematics.
099

Why use HDR Wheels for balancing rather than standard Primaries?

  1. HDR Wheels are faster.
  2. You can pull Lights down to recover a blown sky while Shadows stays untouched — surgical control standard wheels can't match cleanly.
  3. They produce better skin tones automatically.
  4. They reduce noise.
100

After balancing, the image should look:

  1. Like the final grade — the look is now built.
  2. Neutral, well-exposed, with full tonal range — a clean starting point for creative work.
  3. Heavily saturated — saturation defines the look.
  4. Identical to the camera's preview.

  4. Background corrections never apply to skin areas.
107

The Vectorscope's "skin tone indicator" line runs diagonally from centre toward approximately:

  2. 11 o'clock — the typical position of healthy human skin tones regardless of ethnicity.

  4. 5 o'clock.

  1. To redo the Global Look.
  2. To push the image one more step further on top of the parallel-mixer's output — final darkening, casts, or curve adjustments.
  3. To apply a LUT.
  4. To render the file.
130

In Studio's native Film Grain effect, "Advanced Controls" allow you to:

  1. Add grain only at specific tonal ranges (e.g., reduce grain in shadows to keep black areas clean).
  2. Animate grain size over time.
  3. Sync grain to audio peaks.
  4. Replace grain with a LUT.

In the free version, film grain is typically added by:

  1. The native Film Grain effect.

132

In the Blur palette (Color page), the Radius slider behavior is:

  1. Always blurs — higher values blur more.
  2. Below 0.5 = sharpen, above 0.5 = blur. Default 0.5 is no effect.

  4. Radius affects only luminance, not chroma.
133

The under-discussed Sharpening control "Scaling" determines:

  1. The output resolution of the sharpen.
  2. Which pixel-size features get sharpened — higher scaling targets larger features (eyes, lips); lower scaling targets fine texture. Pushing scaling toward 1.0 produces strikingly detailed images for social media.
  3. The Resolve render priority.
  4. The colour space of the sharpen filter.

  4. Crop the timeline to 2.39:1 in Project Settings.

  1. Mask noise that grading exposed.

  3. Reduce file size.
  4. Prevent banding.

  1. The grain has no parameters.

  4. Grain only works in HDR projects.
